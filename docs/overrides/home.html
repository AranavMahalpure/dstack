Cost-effective ML development in the cloud

dstack is an open-source tool designed for ML teams to automate running dev environments and tasks in your cloud.

dstack simplifies the developer workflow of ML teams while driving productivity and reducing cloud costs.

Docs Join community

Dev environments in your cloud on demand

Provision dev environments with a single command

Configure the hardware resources you need (GPU, memory, etc.) and indicate whether you want to use spot or on-demand instances.

Run the configuration with a single command.

dstack will automatically provision cloud resources, fetch your code, and forward ports for secure and convenient access.

Work with dev environments via a desktop IDE

Click the SSH URL from the output to automatically open the cloud dev environment in your local desktop IDE.

The cloud dev environment is automatically pre-configured with the Python environment, and IDE extensions for working with Python and Jupyter notebooks.

Run tasks in your cloud, easily and cost effectively

Run any type of ML task with a single command

Whether you want to run a training script, a data processing job, or a web-based app, simply define its configuration and run it with a single command.

dstack handles the automatic provisioning of cloud resources, including waiting for spot instances if required, as well as port forwarding for secure access.

Use dstack's Python API to save and load artifacts (e.g. model checkpoints and data.)

Manage projects and control costs across teams via UI

Manage users and cloud credentials in a secure and centralized manner.

Browse and manage runs and artifacts through a web-based dashboard.

Track cloud costs per run and manage usage quotas per team via UI.

Run ML on any cloud easily and cost-effectively

Declarative configurations

Define dev environments and tasks declaratively as code. Run them on demand using the CLI, eliminating the need of using cloud UI and proprietary APIs, which can be a nightmare.

GPU cost optimization

Save costs by effectively utilizing cloud spot instances across multiple regions. Automatically recover from interruptions and track GPU usage, while enforcing limits per team.

Data-centric approach

Run dev environments and tasks where your data is located. Eliminate costs associated with ingress and egress data transfers. Ensure the security and privacy of your data and models.

No MLOps is required

dstack integrates with your cloud natively and doesn't require engineering. No hustle with Terraform, Kubernetes or custom Docker images.

Cloud-agnostic

Use the same interface to work with any major cloud provider, including AWS, GCP, and Azure. Use multiple clouds to optimize costs.

Self-hosted

Run the dstack server on your local machine or easily deploy it, as a Docker image, anywhere within your environment, including your own cloud.

Use it easily with your cloud account

Install dstack in under a minute

$ pip install "dstack[aws,gcp,azure]"
$ dstack start

And you're done! Now you can run dev environments and tasks both locally and in your cloud.
